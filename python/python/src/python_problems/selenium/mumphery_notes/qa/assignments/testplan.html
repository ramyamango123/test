Test Plan Assignment

  1. Write a Test Design Specification for the De Anza page(s) you've been assigned. Use the IEEE template on page 184 of your text.

    Student TDS from Fall 2007

  2. Create a Test Case Specification spreadsheet similar to that shown on page 191 of your text only with these four column headers:

    • Test Cases (test case ids from TDS)
    • Actions (a numbered list of instructions to the tester)
    • Expected Results (a description of passing results)
    • Notes (Optional)
    • Actual Results (Optional! Filling out this column is the equivalent of producing a test log or test report, i.e., it's not part of a TCS.)

    The spreadsheet should include every test case you've described in the Test Identification section of your TDS.

    The info in the spreadsheet should be much more detailed than the one-line description of each test case that appears in the Test Identification section of your TDS document. The TDS contains high-level descriptions of each test case, whereas the spreadsheet contains explicit instructions and detailed descriptions.
