  • Bugzilla

    • An open-source (Mozilla) defect-tracking system, used by many companies.

    • Can optionally have Testopia installed "on top of" it. (Testopia is a Test Case Management tool.)

    • Two different Bugzilla installations will be used in this class:

      1. Bugzilla@Mozilla. We'll use this version for two purposes:

        • Filing bugs we find in Testopia's tip. This will come much later in the course, so you do NOT need an account on this Bugzilla installation yet.

        • Learning how to search for Bugzilla bug reports. No account is needed just to search Bugzilla.

      2. The "Testopia tip" installation set up just for this class. We'll use this version for filing ALL bugs other than those we find in Testopia (much later on). So, you DO need an account on this Bugzilla installation asap!

    • The lifecycle of a Bugzilla report

    • Bugzilla demos

      • Logging on

      • Searching for all bugs in a particular product

      • Examining an individual bug report from a SERP (SEarch Results Page)

      • Searching for an individual bug via bugid

      • Searching for bugs via other fields

      • Searching to see if a problem is already in Bugzilla. (Why is this a good idea?)

      • Searching for a list of bugids

      • Saving searches (all, all resolved, all verified, all filed by me, all bugs to cover at next Bug Club meeting, etc.)

      • Changing the field of an existing bug report

      • Setting up a new product/component

      • Adding an attachment to a bug report

      • How to obtain more than two levels of hierarchy

      • Filing a new bug

      • Voting
