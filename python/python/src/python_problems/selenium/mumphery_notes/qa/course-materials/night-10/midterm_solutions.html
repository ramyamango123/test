  1. Which of the four level test plans should be written first?

    1. Acceptance
    2. Integration
    3. System
    4. Unit
    5. It doesn't matter that much, according to the text's authors.
    SOLUTION: (a) Acceptance

  2. What is the order in which the various testing levels are performed?

    1. Integration, Unit, System, Acceptance
    2. Unit, Integration, System, Acceptance
    3. Acceptance, System, Integration, Unit
    4. Unit, Integration, Acceptance, System
    5. Integration, Unit, Acceptance, System
    SOLUTION: (b) Unit, Integration, System, Acceptance

  3. Which of the four testing levels should be done by a group of developers?

    1. Acceptance
    2. Integration
    3. System
    4. Unit
    5. It doesn't matter that much, according to the text's authors.
    SOLUTION: (b) Integration

  4. Which of the four testing levels should have the most realistic testing environment?

    1. Acceptance
    2. Integration
    3. System
    4. Unit
    5. It doesn't matter that much, according to the text's authors.
    SOLUTION: (a) Acceptance

  5. Which of the four testing levels is the least commonly done, according to the authors?

    1. Acceptance
    2. Integration
    3. System
    4. Unit
    5. The authors didn't make such a claim.
    SOLUTION: (d) Unit

  6. Which of the four testing levels is sometimes done at the customer site?

    1. Acceptance
    2. Integration
    3. System
    4. Unit
    5. None of the above.
    SOLUTION: (a) Acceptance

  7. Which of the four testing levels should be done by a single developer?

    1. Acceptance
    2. Integration
    3. System
    4. Unit
    5. None of the above.
    SOLUTION: (d) Unit

  8. Which of these types of tests should be automated first?

    1. Acceptance
    2. Alpha
    3. Regression
    4. Smoke
    5. Usability
    SOLUTION: (d) Smoke

  9. What process utilizes KLOC?

    1. Code coverage
    2. Defect density
    3. Defect seeding
    4. Requirements coverage
    5. Mutation analysis
    SOLUTION: (b) Defect density

  10. Which type of testing is best for uncovering coincidental correctness?

    1. Ad hoc testing
    2. Behavioral testing
    3. Black-box testing
    4. Random testing
    5. White-box testing
    SOLUTION: (e) White-box testing

  11. Which black-box testing technique involves circles and arcs?

    1. Boundary value analysis
    2. Decision tables
    3. Equivalence classes
    4. Orthogonal arrays
    5. State-transition diagrams
    SOLUTION: (e) State-transition diagrams

  12. Which black-box testing technique is especially suited for determining which browser/OS/plug-in combinations to test?

    1. Boundary value analysis
    2. Decision tables
    3. Equivalence classes
    4. Orthogonal arrays
    5. State-transition diagrams
    SOLUTION: (d) Orthogonal arrays

  13. Which black-box testing art is sometimes called monkey testing?

    1. Ad hoc testing
    2. Exploratory testing
    3. Random testing
    4. Semi-random testing
    5. None of the above
    SOLUTION: (c) Random testing

  14. Which type of test involves providing invalid input in the expectation of receiving an error message?

    1. Alpha test
    2. Beta test
    3. Negative test
    4. Positive test
    5. Smoke test
    SOLUTION: (c) Negative test

  15. Which type of black-box art involves concurrent test design and execution?

    1. Ad hoc testing
    2. Exploratory testing
    3. Random testing
    4. Semi-random testing
    5. None of the above
    SOLUTION: (b) Exploratory testing

  16. Which type of measure is used to determine the number of lines of code that are invoked by a set of tests?

    1. Branch coverage
    2. Cyclomatic complexity
    3. Decision coverage
    4. Path coverage
    5. Statement coverage
    SOLUTION: (e) Statement coverage

  17. Which type of data is the most realistic?

    1. Captured
    2. Generated
    3. Manually created
    4. Production
    5. Random
    SOLUTION: (d) Production

  18. Which level of testing is associated with buddy testing?

    1. Acceptance
    2. Integration
    3. Master
    4. System
    5. Unit
    SOLUTION: (e) Unit

  19. Which of the following types of testing tasks is NOT a prime candidate for automation?

    1. Code coverage
    2. Load tests
    3. Performance tests
    4. Smoke tests
    5. Usability tests
    SOLUTION: (e) Usability tests

  20. What is shelfware?

    1. Testing software that can be purchased right off the shelf.
    2. Testing software that gets little use - it just sits on the shelf.
    3. Testing software that requires little maintenance - it can sit on the shelf between product releases.
    4. Testing software developed by QA engineers for their own product, which winds up being sold as a test product (off the shelf).
    5. None of the above.
    SOLUTION: (b) Testing software that gets little use

  21. Which type of tests are run after changes are made in order to ensure that the rest of the system still works correctly?

    1. Confirmation tests
    2. Regression tests
    3. Repetitive tests
    4. Smoke tests
    5. None of the above
    SOLUTION: (b) Regression tests

  22. After a bug in Bugzilla has been fixed, what state would that bug normally be in?

    1. ACCEPTED
    2. FIXED
    3. RESOLVED
    4. VERIFIED
    5. None of the above
    SOLUTION: (c) RESOLVED

  23. After a QA engineer has determined that the bug fix has indeed been made, what Bugzilla state would that bug be moved to?

    1. ACCEPTED
    2. CLOSED
    3. FIXED
    4. RESOLVED
    5. VERIFIED
    SOLUTION: (e) VERIFIED

  24. Which of the following Bugzilla fields cannot be changed?

    1. Priority
    2. QA contact
    3. Severity
    4. Summary
    5. None of the above
    SOLUTION: (e) None of the above

  25. The Waterfall model of software development is an example of a(n) __________.

    1. Extreme model
    2. Iterative model
    3. Random model
    4. Semi-random model
    5. Sequential model
    SOLUTION: (e) Sequential model

  26. Craig and Jeskiel claim that the STEP methodology can transform the Waterfall model into a(n) ____________.

    1. Extreme model
    2. Iterative model
    3. Random model
    4. Sequential model
    5. Semi-random model
    SOLUTION: (b) Iterative model

  27. What name does STEP use for the major testing products such as test plans, scripts, test cases, etc.?

    1. Alphaware
    2. Betaware
    3. Shelfware
    4. Templates
    5. Testware
    SOLUTION: (e) Testware

  28. The STEP relies heavily on document standards developed by what organization?

    1. ACM
    2. CCB
    3. IEEE
    4. ISO
    5. STAR
    SOLUTION: (c) IEEE

  29. What is the name for a defect that has not yet caused a problem because the exact set of conditions has never been met?

    1. Latent defect
    2. Masked defect
    3. Regressive defect
    4. Unmasked defect
    5. None of the above
    SOLUTION: (a) Latent defect

  30. Which two items are assigned H/M/L ratings in steps 3 and 4 of a formal risk analysis?

    1. Complexity and usability
    2. Feature priority and business function priority
    3. Impact and likelihood
    4. Number of new features and number of bugs found in previous release
    5. None of the above
    SOLUTION: (c) Impact and likelihood

  31. What type of distribution involves 80% of the contribution coming from 20% of the contributors?

    1. Boehm
    2. Cem
    3. CMM
    4. Gantt
    5. Pareto
    SOLUTION: (e) Pareto

  32. Code coverage is especially useful in helping locate __________

    1. Dead code
    2. Latent defects
    3. Masked defects
    4. Performance issues
    5. Seeded defects
    SOLUTION: (a) Dead code

  33. What type of non-testing activity should be covered in the Master Test Plan?

    1. Boundary value analysis
    2. Code coverage analysis
    3. Edge case analysis
    4. Pareto analysis
    5. Reviews
    SOLUTION: (e) Reviews

  34. Test data that is quickly outdated due to a rapidly-changing business environment is said to be __________.

    1. Dynamic
    2. Fragile
    3. Glassy
    4. Unstable
    5. None of the above
    SOLUTION: (b) Fragile

  35. Other applications that reside on the same platform as the software being tested is called __________.

    1. Cohabiting software
    2. Coinhibiting software
    3. Collaborative software
    4. Cooperative software
    5. Coresiding software
    SOLUTION: (a) Cohabiting software

  36. Which of the following is in a separate room from the others in a usability lab?

    1. Cameras
    2. Computer
    3. Observers
    4. Prototype
    5. User
    SOLUTION: (c) Observers

  37. Which of the following types of testing is done only at the customer site?

    1. Alpha
    2. Beta
    3. Confirmation
    4. Smoke
    5. Regression
    SOLUTION: (b) Beta

  38. A table whose leftmost column contains features, and which has every other column labeled with a test case identifier, is known as a(n) __________.

    1. Decision table
    2. Gantt chart
    3. Orthogonal array
    4. Pareto analysis
    5. Requirements traceability matrix
    SOLUTION: (e) Requirements traceability matrix

  39. Which of these release strategies involves shipping the entire product to a few customers at a time?

    1. Graded implementation
    2. Gradual implementation
    3. Parallel implementation
    4. Phased implementation
    5. Pilot implementation
    SOLUTION: (b) Gradual implementation

  40. Which of these release strategies involves shipping the entire product to one or a very few number of sites initially?

    1. Graded implementation
    2. Gradual implementation
    3. Parallel implementation
    4. Phased implementation
    5. Pilot implementation
    SOLUTION: (e) Pilot implementation

  41. Which of the four levels of testing frequently requires scaffolding code?

    1. Acceptance
    2. Integration
    3. System
    4. Unit
    5. None of the above
    SOLUTION: (b) Integration

  42. What is the name for the ratio of defects to size?

    1. Defect density
    2. Defect dispersion
    3. Defect scope
    4. Defect seeding
    5. None of the above
    SOLUTION: (a) Defect density

  43. Which of the following is NOT another name for white-box testing?

    1. Behavioral testing
    2. Clear-box testing
    3. Glass-box testing
    4. Translucent-box testng
    5. Structural testing
    SOLUTION: (a) Behavioral testing

  44. Which black-box testing technique lumps tests together when the following conditions are met: they all test the same thing; if one catches a bug, the others probably do too; and if one doesn't catch a bug, the others probably won't either?

    1. Boundary value analysis
    2. Decision tables
    3. Equivalence classes
    4. Orthogonal arrays
    5. State-transition diagrams
    SOLUTION: (c) Equivalence classes

  45. What activity is associated with a "cut line?"

    1. Code coverage
    2. Confirmation testing
    3. Defect seeding
    4. Mutation analysis
    5. Software risk analysis
    SOLUTION: (e) Software risk analysis

  46. The entire product to be tested, including software and associated documentation, has as its acronym __________.

    1. CCB
    2. CMM
    3. STEP
    4. SUT
    5. ULAB
