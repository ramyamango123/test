CIS-200W - Automated Web Page Testing with Selenium IDE

De Anza College

Spring 2008

Project

Use SIDE to automate the Test Case Specification below. You should create a single test suite, composed of several test cases, one for each of the individual test cases below. Note that many of the test cases contain more than one expected result; for these test cases, it is fine to have the multiple verifications inside a single test case file.

The search string to be used for each test case should appear once and only once! You should store the search string's value in a variable, and use the variable in all future commands that need it. If you need to modify the search string somewhat for use as a regular expression, store that modified value in a second variable, and use it in all future commands that need it.

All test cases should be started from http://www.deanza.edu/searchcenter.

HINT: Test cases #1 and #3 are fairly similar. Both require using the flow control extension. They're also both harder than the other problems, so I'd suggest doing them last.

Test ID Action Expected Results Hints
SC-1 Word + quoted phrase search of entire site
  1. Enter pass "no pass" into the textarea just below ...search the entire De Anza Web site.
  2. Select the Search button.
  • The browser title should be Search Results: pass "no pass".
  • The phrase Searched for pass "no pass" string should appear on the page.
  • The phrase Results of 1 - 10 of about should appear on the page.
  • The textarea above the search results should contain the original search string, including quotes.
  • Each of the 10 resulting snippets should include the word pass and the phrase no pass, although the punctuation and spacing may vary.
  • One or more while/endWhile constructs from the SIDE flow control extension are probably going to be necessary.
  • Indented snippets are HTML blockquotes, whereas non-indented snippets are HTML paragraphs.
SC-2 Search must be case-insensitive
  1. Enter pass "no pass" into the textarea just below the search the entire De Anza Web site.
  2. Select the Search button.
  3. Note the number of hits on the search results page.
  4. Enter PASS "NO PASS" into the textarea just below the search the entire De Anza Web site.
  5. Select the Search button.
  6. Note the number of hits on the search results page.
The number of hits obtained from the uppercase search should be exactly the same as from the lowercase search.
SC-3 Basic one-name search of Faculty site
  1. Enter may-pumphrey into the textarea underneath Faculty Web Sites.
  2. Select the Search button.
  • The browser title should be Search Results: may-pumphrey.
  • The phrase Searched for may-pumphrey string should appear on the page.
  • The textarea above the search results should contain the original search string, including the hyphen.
  • Each of the 10 snippets should include the URL domain faculty.deanza.edu/.
SC-4 Double-quoted single-word search should yield same results as unquoted single-word search
  1. Enter may-pumphrey into the textarea underneath Faculty Web Sites.
  2. Select the Search button.
  3. Note the number of hits on the search results page.
  4. Enter "may-pumphrey" (including double quotes) into the textarea underneath Faculty Web Sites.
  5. Select the Search button.
  6. Note the number of hits on the search results page.
The number of hits obtained from the quote-free search should be exactly the same as from the quoted search.
SC-5 Single-quoted single-word search should yield same results as unquoted single-word search
  1. Enter may-pumphrey into the textarea underneath Faculty Web Sites.
  2. Select the Search button.
  3. Note the number of hits on the search results page.
  4. Enter 'may-pumphrey' (including single quotes) into the textarea underneath Faculty Web Sites.
  5. Select the Search button.
  6. Note the number of hits on the search results page.
The number of hits obtained from the quote-free search should be exactly the same as from the quoted search.
SC-6 Negative: Search for nonexistent string
  1. Enter NonexistentFirstName NonexistentLastName into the textarea underneath Faculty Web Sites.
  2. Select the Search button.
  • The browser title should be Search Results: NonexistentFirstName NonexistentLastName.
  • The textarea above the search results should contain the original search string.
  • The search results page should contain the following error message: 
    
Your search - NonexistentFirstName NonexistentLastName- did not match any documents.
No pages were found containing "NonexistentFirstName NonexistentLastName".
SC-7 Basic check of Cached links
  1. Enter loan into the textarea just below Financial Aid.
  2. Select the Search button.
  3. Look for a Cached link in the SERP.
If a Cached link does not exist, output an error message to the log file and quit the test
else
  1. Select the Cached link.
  2. Capture the title of the target landing page.
  3. Return to the SERP.
  4. See if the title of the target landing page is present on the SERP (it would be a snippet title if present). If it is present, that means the Cached link takes one to one of the hits' URLs, which gives one reasonable confidence that the Cached links are working. If the title of the target landing page is not present on the SERP, the test should fail.

NOTE: It would be better but much harder to iterate through every hit on the SERP, capturing the title of each, and then seeing if the Cached link led one to a page with the same title.
