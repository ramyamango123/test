  2. Regular Expressions: The Character Class with a Range

    A hyphen (dash) can be used between two characters within a character class to create a range.

    Ranges help make regular expressions shorter, and hence, easier to read than they would be otherwise.

    For example, both of these two ranges match any alphabetical character, whether uppercase or lowercase:

    • [abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ]
    • [a-zA-Z]

  4. Regular Expressions: The Numeric Quantifier

    The syntax is {n}, where n is the number of occurrences of the immediately preceding entity (character or range or group) which one wants to match.

    For example, [a-zA-Z]{10} will match ten consecutive alphabetical characters, whether uppercase or lowercase.

    [aeiouAEIOU]{4} will match 4 consecutive vowels, regardless of case.

  5. Regular Expressions: Negation

    A caret (^) at the beginning of a character class negates that character class, meaning that any character not inside the square brackets will be matched.

    For example, [^a-zA-Z]{10} will match any ten consecutive non-alpha characters.

  6. Regular Expressions: The "dot" (period)

    A period (.) can be used to match any single character (including an actual period).

  7. Regular Expressions: The Asterisk Quantifier

    An asterisk (*) is another type of quantifier, which matches 0 or more of the preceding character or group.

    Taken together, .* will match 0 or more of any character, or in short "anything or nothing." .* is used very often within larger regular expressions.
