Globbing, Exact, and Prefix-Free Patterns

  1. Question: When does one use globbing, exact, regular expression (covered next week), and prefix-free patterns?

    Answer: Whenever the reference documentation for a Selenium command specifies a pattern argument.

    Example: verifyTitle

  2. Globbing

    1. Globbing is sometimes called "filename expansion."

    2. Globbing utilizes "wildcards."

    3. Globbing and regular expressions ARE NOT THE SAME THING!!!

    4. The usual globbing wildcards are:

      1. ? - matches any single character. Thus, a?b would match "aXb" or "a b".

      2. * - matches anything, including nothing. Thus, a*b would match:

        • ab
        • aXb
        • a Now is the time for all good women to come to the aid of their countryb
        • <an infinite number of other strings>

      3. [] - character class, which matches any single character that is included between the brackets. Thus, [0123456789] would match any digit.

        [abcdefghijjklmnopqrstuvwxyz] would match any lowercase alphabetic character.

        [A-Za-z0-9] would match any alphabetical letter, regardless of case, or any digit.

    5. To use globbing within a Selenium IDE test, one must prefix a pattern with glob:.

    6. Note that the Selenium reference docs make it sound like only the ? and * work. My experiments reveal that this is indeed the class, i.e., character classes are NOT supported for glob: patterns.
