TestRunner

  1. Creating and executing a test suite via Selenium TestRunner

    1. Create an HTML file with a one-column table in the same directory as the test cases.

    2. Fill in each cell of the table with a link to a testcase.html file.

      Sample test suite. Do a View=>Page Source in order to see the raw HTML needed to create a test suite suitable for TestRunner.

    3. Point Firefox browser at the following URL with baseURL and parameters modified as appropriate.

      chrome://selenium-ide/content/selenium/TestRunner.html?baseURL=http://www.deanza.edu&test=file:///Users/mamp/Desktop/Selenium/Night-02/testsuite.html&auto=true

      auto=true causes TestRunner to immediately run the test suite. Omitting this parameter will eliminate this automatic execution.

    LAB EXERCISE TESTRUNNER-1

    1. Using Selenium IDE and any four links from the De Anza directory you wish, create four separate tests, one per link (as opposed to one test covering four links as you did for Lab Assignment-1). Run each test after you've created it to make sure it works.

    2. Create a test suite to execute each of your four tests in turn.

    3. Use a chrome URL to bring up TestRunner.

    4. Execute your test suite via TestRunner, making sure that all your tests pass or fail as expected.
