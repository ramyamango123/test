    I think the glob: one is best for the following reasons:

    • It's better than its unprefixed functional equivalent because the glob: prefix adds to the readability of the test case.

    • It's better than the exact: one because it allows for minor punctuation and spacing changes to be made between words without the test case failing.

    While it's true that a malicious programmer could insert all sorts of nasty words between the title words without the glob: test catching it, that's a corner case. Our primary focus should be on creating tests that are as robust as possible, while still checking basic correctness.
