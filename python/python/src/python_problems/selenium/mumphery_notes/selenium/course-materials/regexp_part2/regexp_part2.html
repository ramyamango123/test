Real-world regular expression usage within Selenium IDE

  1. Negative test example: Recording clicking on a form button when not enough selections have been made, then verifying presence of error message on response page via verifyTextPresent.

    1. Both open and verifyTextPresent were selected from context menu.

    2. assertTextPresent could be used instead, in which case the test would stop executing if the error message text was not found on the response page.

    3. Note the argument to clickAndWait. This is an Xpath argument. Xpath will be covered in an upcoming lecture.

  2. Negative test example: Record clicking on a form button when no selections have been made, then verify presence of error message on an alert via assertAlert.

    1. The entire test was created with the Record feature.

    2. Note that the alert does not appear when the test is played back.

    3. To prove this test works, manually modify the argument to assertAlert to contain at least one additional/missing/different character.

      It's always a good idea to test your test to make sure that it works. Force a passing test to fail via minor tweaking of the Selenese code. And if possible, force a failing test to pass via minor tweaking of the Selenese code. Disallow coincidental correctness as much as possible!

    4. verifyAlert could be used instead, in which case the test would continue executing even if the alert's text was incorrect (although an error message would still appear in the log file).

  3. Positive test example.

    1. Note the select command, used to select a radio button.

    2. Note the type command, used to enter input from the keyboard.

    3. Note that three separate verify commands were needed to thoroughly test that the search results were correct.
