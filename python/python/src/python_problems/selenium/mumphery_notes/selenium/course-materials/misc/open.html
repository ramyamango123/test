Opening a Page

  1. Opening a Page: Base URL

    • Saved as part of test as of 1.0 Beta; was saved in user's cookie in 0.8.

    • 1.0 Beta tests that rely on it are harder to read (because URL to page being tested is spread across two locations within test program).

    • Buggy! For example, changing the Base URL within the Source tab doesn't have any effect on the Base URL displayed at the top of the SIDE GUI. So one can have two different values on display for Base URL at the same time for the same test!

  2. Opening a Page: First argument to open command

    • Can be absolute (complete) URL, e.g.,
      http://www.deanza.edu/directory/dir-az.html

    • Can be relative (in relation to Base URL), e.g.,
      /directory/dir-az.html OR...
      directory/dir-az.html

  3. Opening a Page: Issue with Slashes (/)

    In 0.8, using a Base URL without a trailing slash, e.g.,
    http://www.deanza.edu
    while also using a relative URL without a leading slash, e.g., directory/dir-az.html
    would result in SIDE not being able to open the page.

    This issue has been fixed in 1.0 Beta.

  4. Opening a Page: Options=>Options...=>Record absolute URL checkbox

    Affects only what will be put into first argument of open command during recording of a test

  5. Recommendations:

    • Avoid Base URL until such time as you want to help the OpenQA team get this feature right OR until you have a compelling reason for using it.

    • Turn Record absolute URL on and leave it on (until you have a compelling reason for doing otherwise).

    • Always provide an absolute URL argument to open (unless you have a compelling reason for doing otherwise).

LAB EXERCISE OPEN-1

Turn on Record absolute URL. Try recording a simple test to verify that an absolute URL is being recorded as the first argument to the open command.
