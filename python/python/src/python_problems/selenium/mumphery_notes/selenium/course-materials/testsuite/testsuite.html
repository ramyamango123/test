Test Suites Using SIDE 1.0 Beta

  1. The good news? Creating and executing a test suite is a lot easier in SIDE 1.0 Beta than in SIDE 0.87.

    The bad news? Creating and executing a test suite in SIDE 1.0 Beta is not quite as easy as the GUI would lead one to believe.

  2. Method-1 (somewhat manual):

    1. Create an HTML file (outside SIDE, using an editor) with a one-column table in the same directory as the test cases.

    2. Fill in each cell of the table with a link to a testcase.html file.

      Sample test suite. Do a View=>Page Source in order to see the raw HTML needed to create a test suite suitable for SIDE.

    3. Select File=>Open Test Suite to open the test suite.

    4. Use the first right-pointing green arrow to execute the test suite.

  3. Method-2 (more automated):

    1. Create 3 simple test cases, one after the other, allowing them to "accumulate" in the Test Case frame on the left of SIDE.

    2. Select File=>Save Test Suite As... to create a test suite that invokes the three test cases.

    3. Close SIDE and bring it up again (just to prove that your test suite works).

    4. Select File=>Open Test Suite to open your Method-2 test suite.

    5. Use the first right-pointing green arrow to execute the test suite.

  4. Method-3 (which doesn't exist as far as I can tell!).

    There ought to be a way to create the one-column table test suite using Table or Source view.

    However, selecting New Test Suite just sets one up to create a new test case, allowing one to use Method-2 eventually.

  5. Method-4 (which doesn't exist either as far as I can tell!).

    Once one has created several different test cases, there ought to be a way to load them into the Test Case frame on the left of SIDE, in the order that one wants, and then select File=>Save Test Suite As... to create a test suite that invokes those test cases in the order they appear.

    However, selecting Open... to open an existing test case always removes whatever test cases are already in the Test Case frame on the left of SIDE.

    What does the absence of Method-3 and Method-4 mean with respect to which of the earlier two methods you'll be using more often?

    LAB EXERCISE TESTSUITE-1

    1. Using Selenium IDE and any four links from the De Anza directory you wish, create four separate tests, one per link (as opposed to one test covering four links as you did for Lab Assignment-1). Run each test after you've created it to make sure it works.

    2. Create a test suite via Method-1 to execute each of your four tests in turn.

    3. Execute your test suite, making sure that all your tests pass or fail as expected.

    4. Repeat the three steps above, this time using Method-2 to create the test suite.

    5. For hand-in purposes, attach one of your two test suites plus the individual test cases called by that test suite.
