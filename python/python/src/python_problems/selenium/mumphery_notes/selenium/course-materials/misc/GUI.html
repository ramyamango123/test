Selenium IDE GUI Features

  1. Executing only a portion of a test

    1. From one command through the end of test...

      1. In Command view, select the command where execution should start.

      2. Bring up the Context menu (alt-click for Windows, CTRL-click for Macs).

      3. Select the Set / Clear Start Point item.

      4. Observe the right-pointing green triangle at the beginning of the selected command.

      5. Select the usual glyph ("Play from the beginning or start point") to start playing the test from that command.

      6. To delete the Start Point, either type an "S" OR re-select the Set / Clear Start Point item from the context menu.

    2. From the beginning through a Breakpoint...

      1. In Command view, select the command where execution should end. NOTE: Do not select a comment! SIDE will allow one to set a breakpoint at a comment, but it will be a non-functioning breakpoint!

      2. Bring up the Context menu.

      3. Select the Toggle Breakpoint item.

      4. Observe the double yellow vertical lines at the beginning of the selected command.

      5. Select the usual glyph ("Play from the beginning or start point") to start playing the test from that command.

      6. To delete the Breakpoint, either type a "B" OR re-select the Toggle Breakpoint item from the context menu.

    3. One command only - Select the Execute this command item from Context menu in Command view.

    LAB EXERCISE GUI-1

    Bring up one of your stored tests that contains several commands. Set a Breakpoint and then execute the test from the beginning to that point. Then try executing one command only from the test. (Warning: Using a Start Point will be more difficult because the open command will not get executed.)

  2. Stepping through a test's execution

    1. Select the usual green right-pointing triangle ("Play from the beginning or start point").

    2. Immediately select the Pause glyph (two vertical yellow bars).

    3. Observe that the right-and-down-pointing blue arrow ("Step") has now become active.

    4. Select this "Step" blue arrow repeatedly in order to step through the execution, one command at a time.

    LAB EXERCISE GUI-2

    Bring up one of your stored tests that contains several commands. Execute this test via stepping through it.

  3. Test names vs. file names

    1. The thead portion of each test contains the name of that test.

    2. The default name of a new test is "New Test."

    3. Once a test has been saved, its name becomes the same as the name of that test's filename, minus the .html extension. However! That change is not visible unless the test is reopened or SIDE is closed and then reopened.

    4. Can a user override that name change with a different name change? Not as far as I can tell! The name supplied by the user is always overwritten by the filename minus the .html extension.
